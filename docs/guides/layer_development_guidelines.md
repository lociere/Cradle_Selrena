# 各层开发指导

此文档为未来开发者编写，按照架构分层提供细致的开发指导。

## 1. 核心基础设施层（core）
- 说明如何编写配置管理、事件总线客户端、AI 服务等基础模块。
- 包括生命周期注册、日志统一、线程/异步注意事项。
- 示例：`config_manager.py`、`event_bus_client.py` 模板。

## 2. 适配器层（adapters）
- 如何实现 `KernelPort`、`MemoryPort` 等接口。
- 范例：编写新的第三方平台适配器（如 Discord、WeChat）。
- 注意保持与协议版本兼容，避免业务逻辑混入。

## 3. 领域层（domain）
- 领域对象定义指南：使用 dataclass + Pydantic 验证。
- 如何扩展人设/记忆/情感模型。
- 测试建议：纯数据模型无需外部依赖。

## 4. 数据模型层（schemas）
- Pydantic 模型写法规范。
- schema 与配置/事件的映射关系。
- 建议添加文档注释与校验逻辑。

## 5. 应用服务层（application）
- 编写用例编排的基本流程（接收、检索、生成、存储、发送）。
- 如何调用 ports、domain 对象。
- 示例：`ConversationService` 扩展步骤。

## 6. 工具层（utils）
- 常见工具函数模式：纯函数、无副作用、易测试。
- 异步工具、路径处理、配置加载等示例。

## 7. 业务核心层（persona/memory/intention/agent）
- 如何将复杂逻辑拆分成可组合的小模块。
- 在此层保持与外界无耦合，所有数据通过参数传递。
- 建议的目录结构与命名规则。

## 8. 推理引擎层（inference）
- 引擎接口规范：`BaseBrainBackend`、异步协议。
- 如何编写新的本地/远程引擎。
- 调度器与路由器设计要点。
